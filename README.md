# architectural-design

UML system design for a **cinema booking application**, modeled across four diagram types using Visual Paradigm.

## Diagrams

| Folder | Diagram Type | Purpose |
|---|---|---|
| `use case` | Use Case Diagram | Actors and system interactions (guest, member, admin) |
| `class` | Class Diagram | Domain model — entities, attributes, relationships |
| `sequence` | Sequence Diagram | Object interaction over time (booking flow, auth, etc.) |
| `activity` | Activity Diagram | Workflow logic and decision branches |
| `TD` | Travaux Dirigés | Lab exercise files |
| `ws1` | Workshop 1 | Workspace scratch files |

## Domain

The modeled system covers a cinema booking platform with typical actors and flows:

- Guest browsing and seat selection
- Member registration, login, and booking management
- Admin management of films, showtimes, and halls
- Payment and ticket generation

## Tooling

Diagrams are in `.vpp` format — open with **[Visual Paradigm Community Edition](https://www.visual-paradigm.com/download/community.jsp)** (free).
